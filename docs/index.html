üìñ Project Overview

This project demonstrates Module Federation using Webpack 5, enabling micro-frontend architecture where multiple independent applications can share code dynamically at runtime.

Architecture

  • Remote Application: Exposes components (Counter, Router, Info) that can be consumed by host apps
  • Host Applications: Consume remote components and integrate them seamlessly
  • Technology Agnostic: Works across React, Angular, and other frameworks
  • Runtime Loading: Components are loaded dynamically at runtime, not build time

Exposed Remote Modules

Module Name Path Description
./Counter ./src/components/Counter.tsx React Counter component with state management
./Info ./src/info.ts Utility module with app information

Loading Remote Components in React

// In host-webpack-react/src/components/RemoteCounter.tsx
import React, { useEffect, useRef } from 'react';
import { loadRemote } from '@module-federation/enhanced/runtime';

const Remote = () => {
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    let mod: any = null;
    loadRemote('remote/Counter')
      .then(({ default: remote }: any) => {
        mod = remote();
        mod?.mount?.(ref.current);
      })
      .catch(() => {
        if (ref.current) {
          ref.current.innerText = 'Error loading remote module';
        }
        console.error('Error loading remote module');
      });
    return () => mod?.unmount?.();
  }, []);
  
  return <div ref={ref}>Loading....</div>;
};

export default Remote;

Loading Remote Components in Angular

// In host-angular/src/app/components/remote/remote.component.ts
import {
  AfterViewInit,
  Component,
  ElementRef,
  OnDestroy,
  ViewChild,
} from '@angular/core';
import { loadRemote } from '@module-federation/enhanced/runtime';

@Component({
  selector: 'app-remote',
  imports: [],
  templateUrl: './remote.component.html',
  styleUrl: './remote.component.css',
})
export class RemoteComponent implements AfterViewInit, OnDestroy {
  @ViewChild('ref', { static: false })
  ref!: ElementRef;
  remoteElement: any;

  ngAfterViewInit(): void {
    loadRemote('remote/Counter')
      .then(({ default: remote }: any) => {
        this.remoteElement = remote();
        this.remoteElement?.mount?.(this.ref.nativeElement);
      })
      .catch(() => {
        this.ref.nativeElement.innerText = 'Error loading remote module';
      });
  }
  
  ngOnDestroy(): void {
    this.remoteElement?.unmount?.();
  }
}
Key Points:
  • Both React and Angular use the same loadRemote() API
  • Remote components expose mount() and unmount() lifecycle methods
  • Use refs/ElementRef to get DOM elements for mounting
  • Always handle cleanup in useEffect return or ngOnDestroy

üéØ How to Use Module Federation

Step-by-Step Guide

1. Understanding the Architecture

Module Federation allows you to:

  • Remote App: Exposes components that other apps can consume
  • Host Apps: Load and use components from remote apps at runtime
  • Dynamic Loading: Components are fetched when needed, not bundled at build time

2. Setting Up Remote Application

The remote application must expose components using Module Federation Plugin:

// remote/webpack.config.common.ts
import { ModuleFederationPlugin } from '@module-federation/enhanced';

new ModuleFederationPlugin({
  name: 'remote',                    // Unique name for this remote
  manifest: true,                    // Generate manifest file
  filename: 'remoteEntry.js',        // Entry file name
  exposes: {
    './Counter': './src/components/Counter.tsx',   // Expose Counter component
    './Info': './src/info.ts',                     // Expose Info module
    './Router': './src/components/Router.tsx'      // Expose Router component
  },
  shared: undefined                  // Share dependencies (optional)
})

3. Creating Exposable Components

Remote components should export mount/unmount functions for lifecycle management:

// remote/src/components/Counter.tsx
import React from 'react';
import ReactDOM from 'react-dom/client';
import Counter from './CounterUI';

export default function () {
  let root: any = null;
  
  return {
    mount: (container: HTMLElement) => {
      root = ReactDOM.createRoot(container);
      root.render(<Counter />);
    },
    unmount: () => {
      root?.unmount();
    }
  };
}

6. Loading Remote Components in React

import React, { useEffect, useRef } from 'react';
import { loadRemote } from '@module-federation/enhanced/runtime';

const RemoteCounter = () => {
  const ref = useRef<HTMLDivElement>(null);

  useEffect(() => {
    let mod: any = null;
    
    // Load remote module
    loadRemote('remote/Counter')
      .then(({ default: remote }: any) => {
        mod = remote();
        mod?.mount?.(ref.current);
      })
      .catch((error) => {
        console.error('Failed to load remote:', error);
        if (ref.current) {
          ref.current.innerText = 'Error loading remote module';
        }
      });
    
    // Cleanup on unmount
    return () => mod?.unmount?.();
  }, []);

  return <div ref={ref}>Loading remote component...</div>;
};

export default RemoteCounter;

7. Loading Remote Components in Angular

import { Component, ElementRef, ViewChild, AfterViewInit, OnDestroy } from '@angular/core';
import { loadRemote } from '@module-federation/enhanced/runtime';

@Component({
  selector: 'app-remote-counter',
  template: '<div #ref>Loading...</div>'
})
export class RemoteCounterComponent implements AfterViewInit, OnDestroy {
  @ViewChild('ref', { static: false }) ref!: ElementRef;
  remoteElement: any;

  ngAfterViewInit(): void {
    loadRemote('remote/Counter')
      .then(({ default: remote }: any) => {
        this.remoteElement = remote();
        this.remoteElement?.mount?.(this.ref.nativeElement);
      })
      .catch((error) => {
        console.error('Failed to load remote:', error);
        this.ref.nativeElement.innerText = 'Error loading remote module';
      });
  }

  ngOnDestroy(): void {
    this.remoteElement?.unmount?.();
  }
}

GitHub Pages Deployment

This project uses GitHub Actions for automated deployment to GitHub Pages.

Workflow File Path

.github/workflows/deploy.yml

Deployment Process

  1. Trigger: Automatically runs on push to main branch or manual workflow dispatch
  2. Build: Builds all three applications (host-webpack-react, host-angular, remote) in parallel
  3. Artifact Upload: Uploads build artifacts for each app
  4. Deploy: Restructures artifacts into final-dist/app-name structure
  5. GitHub Pages: Deploys to GitHub Pages

Manual Deployment Workflow

You can trigger deployment manually from GitHub Actions tab:

  1. Go to your repository on GitHub
  2. Click "Actions" tab
  3. Select "Deploy App to GitHub Pages" workflow
  4. Click "Run workflow"
  5. Select which app to deploy (or "all")
  6. Click "Run workflow" button
Automatic Deployment: Every push to main branch automatically triggers deployment of all apps.

üîß Troubleshooting

Common Issues

Issue: Remote module not loading

Solution: Ensure the remote application is running before starting the host application. Check that REMOTE_HOST environment variable is correctly configured.

Issue: CORS errors in development

Solution: The remote application must allow CORS from the host application origin. Check webpack dev server configuration.

Issue: Module not found errors

Solution: Verify the exposed module names match exactly in both remote and host configurations. Check the manifest file is being generated correctly.

Development Tips

  • Always start the remote application first
  • Use browser DevTools Network tab to verify remote module loading
  • Check console for Module Federation specific errors
  • Ensure all dependencies versions are compatible
  • Use npm run build to test production builds locally before deployment
